# サイトコンフィグ & スタイルガイド作成

`cmx/site-config.md`（開発向け）と `workflows/style-guide.md`（ライター向け）を対話的に作成します。

## 手順

### 1. ヒアリング

以下を順に質問する:

**サイト基本情報:**
1. **どんなサイトを作りますか？**（種別・目的）
   - コーポレート / ブログ / LP / EC / ドキュメント / メディア / その他
2. **サイト名は？**
3. **想定ユーザーは？**（ターゲット層）

**デザイン方針:**
4. **サイトの雰囲気を一言で表すと？**（例: プロフェッショナル、カジュアル、モダン）
5. **デザインの方向性は？**（カラー、フォント、レイアウトの好み）
6. **参考にしたいサイトはありますか？**（URL）

**執筆ルール:**
7. **文体は？**（例: ですます調、である調）
8. **読者をどう呼びますか？**（例: みなさん、あなた、お客様）
9. **避けるべき表現はありますか？**（例: カジュアルすぎる表現、特定の用語）
10. **ブランド用語の表記ルールはありますか？**（サービス名、製品名等の正しい表記）

**制約:**
11. **絶対に避けたいことはありますか？**（デザイン、トーン、表現等）
12. **コンテンツに関するルールはありますか？**（画像ルール、CTA ルール等）

回答が曖昧な場合は具体例を挙げて確認する。
「特にない」という回答は空欄のままにする（後で `/maintain/config` で追加可能）。

### 2. ファイル生成

ヒアリング結果をもとに **2つのファイル** を生成する:

**`cmx/site-config.md`** — 開発者・AI エージェント向け
- 基本情報、ターゲット、トーン・雰囲気、デザイン方針、やってはいけないこと、コンテンツルール
- 既存のテンプレート構造を維持すること

**`workflows/style-guide.md`** — ライター向け
- 基本情報（言語、文体、一人称、読者呼称）
- トーン・ボイス（全体のトーン、避けるべき表現、推奨する表現）
- ブランド用語集
- 禁止事項
- フォーマットルール・記事構成・SEO考慮事項はテンプレートのまま維持

ヒアリングの回答を **両方のファイルに適切に振り分ける**:
- デザイン関連（カラー、フォント、レイアウト）→ site-config.md のみ
- 執筆関連（文体、トーン、用語）→ 両方（site-config.md のトーンセクション + style-guide.md）
- 禁止事項 → 両方

### 3. 確認

生成した2ファイルをユーザーに提示し、修正があれば反映する。

## 出力

```
サイトコンフィグとスタイルガイドを作成しました:
- cmx/site-config.md（開発方針）
- workflows/style-guide.md（執筆ルール）

次のステップ: /setup/02_init で環境をセットアップしましょう。
```
